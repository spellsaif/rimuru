# Security & Isolation

The cornerstone of Rimuru's safety is the **Sovereign Blindspot**. This ensures that the AI assistant remains "blind" to your most sensitive secrets, even while it has the power to manage your code.

## The Sovereign Blindspot

This is a hard-coded blockade within the file-handling engine that ensures the AI can never "see" its own session keys or your encrypted Vault.

### Path-Level Isolation
The file-system Runes (`readFile`, `listDir`, `editFile`) are hard-coded to reject any path inside the `.rimuru/` directory. 
- **WhatsApp Protection**: The session keys for your WhatsApp connection live in `.rimuru/circles/whatsapp`. The AI can never read these.
- **Vault Protection**: The AI can never see the `vault.json` file.

## The Permission Vows

Runes are classified by risk levels. Every interaction is gated by a **Vow Check**.

| Risk Level | Description | Example Tools |
| :--- | :--- | :--- |
| **`read`** | Safe. Reading files or listing directories. | `readFile`, `listDir` |
| **`write`** | Moderate risk. Editing files or committing to Git. | `editFile`, `gitCommit` |
| **`execute`** | High risk. Running terminal commands or scripts. | `runCommand`, `bash` |
| **`network`** | High risk. Making external API calls beyond the LLM shard. | `curl`, `fetch` |

## The Fortress: Encrypted Vault Architecture

Security is not just about blocking access; it's about robust encryption. The Rimuru Vault is a sovereign fortress for your API credentials.

### Encryption Standards
- **Algorithm**: AES-256-GCM (Galois/Counter Mode).
- **Key Derivation**: We use Argon2 for high-memory-cost key derivation, protecting your master key from brute-force attacks.
- **Storage**: Secrets are stored in `vault.json`, but they are never readable without the runtime's cryptographic context.

### Access Control
When a Rune needs an API key (e.g., to talk to Anthropic), it doesn't read the key itself. Instead, it requests a **Secret Resolve** from the Vault. The Vault verifies the Rune's risk profile and only then provides the decrypted token for the duration of the network request.

## Redacted Audit Traces

Rimuru generates `traces` (JSON files) for every interaction. These are vital for debugging but could contain secrets. Rimuru's tracing engine automatically redacts:
- Any string matching an entry in your **Vault**.
- Any absolute file paths that reveal your system's username.
